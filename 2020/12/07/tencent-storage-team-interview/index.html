写在魔幻年的腾讯面试笔记

2020 年是魔幻的一年。上半年，新冠席卷全球。但也许是运气使然，毕业前这半年的动乱，似乎没有对我的生活造成什么大影响。然后，想不到的是这运气在正式毕业之际用尽了。某种意义上，我也被疫情波及了，事业迟迟没法走向正轨。

想不到，还没入职前司，就决定离开了。

其实现阶段专心做点事情比浮躁的去投简历好得多，只是没办法，所以通过工作室在 PCG 的小伙伴勾搭上了在 TEG 的泥电学长，把简历推了过去（感谢学长的内推 OwO）。学长超随和的，表示简历和他们比较 Match。因为写底层代码，组里对能力要求不低，建议多刷刷题，而且对于写到简历上的东西深度一定要有，也给了不少其他的建议。

然后就有了这次面试。

一面

  • 自我介绍？

    • 简单说了两句，发现没什么好介绍的。

  • …………那？来讲讲你的大学期间做的事？（也许是期待我也是个 ACM 选手吧 0_0）

    • 开始的时候玩硬件，后来发现还是写代码香，说了下工作室和我们负责的 校内 BBS，还有倒腾裸金属服务器的故事….Blablabla

  • 问简历

    • 闲聊 Kubernetes
      • 搭过 K8S 吗？在学校有把应用跑上 K8S 吗？
        • 搭过，但没人用，后来就没维护了。然后总结了下教训： K8S 很重，必须要业务系统复杂度到了一定程度或者有了一定的规模，上 K8S 才能起到降低各种成本和负担的效果。（面试官表示认同）
      • 看你写了好多 Kubernetes 的东西。说说 Kubernetes 有哪些基本的组件，各自的作用是什么？
      • （打断）你说到 kube-apiserver 把状态存在 etcd，etcd 中比较重要的分布式算法是什么。 “Raft”
      • 介绍一下 Raft？你觉得 Raft 解决的最关键的问题是什么？（问得非常符合我的预期 🐶）
        • 最关键的是分布式系统的共识问题，各节点如何就某个问题或选择达成共识。blablabla…..
        • （打断）你提到了 Paxos。能讲讲它和 Raft 的区别吗？（不能）委婉表示不是很了解 Paxos。
        • 好。那就继续讲讲 Raft 吧。
        • 说了一下 Raft 的几个阶段
      • kube-controller-manager 主要实现些什么？为什么有 kube-apiserver 还不够？
        • 表示前者在基于后者定义的资源，实现了更多 PaaS 的能力。比如 DaemonSet …. Blablabla….
      • PaaS 主要应该提供哪些能力？
        • 没怎么总结过。从资源调度和弹性计算的角度讲了一下（提供无感知的水平缩扩容），糊弄了过去。（后来想想，服务发现、流量调度、服务状态的遥测、降级熔断这些 PaaS 能力都可以讲一讲，进而也可以上升到对业务系统的价值层面做总结）
      • 容器的隔离怎么做的？需要隔离哪些东西？网络、计算都是隔离的。基本上用的 Linux Namespace。（严格来说不严谨，面试结束后想到了 Kata 的玩法）
      • 讲讲你了解的 Namespace 有哪些。简单讲了 pid、mount、network 各自的作用。
      • Kubelet 拉起容器的过程？会不会有顺序的问题？表示没看过这部分代码。
        • 那说说让你来设计这块，你会怎么做。
      • Pod 和容器的区别？blablabla….
      • 容器生命周期？表示没看过这部分代码。状态就没完整背过。所以简单说可能怎么设计，可能会有哪些状态。
    • 可以了。我们聊一聊别的。
      • 不继续聊聊 Raft 吗🐶？不了不了，这个拓展起来能聊一天（笑）。我：确实能聊一天（笑）。
    • 问实习工作。头条那边主要玩 ES，为什么要改打分规则。（挖坑开始 - -）
      • 简单介绍了实习期间做的事。因为给业务系统加拼音缩写查询，改了分词，必须微调打分排序规则。表示做的都是微不足道的事。
      • 问题：让你设计一个大规模的搜索系统，分布式下，正排、倒排数据怎么放？
      • 问题：如果你写的这个系统要升级版本，怎么保证不影响线上服务的情况下最快完成升级。
        • 答得不好，给的思路是在保证数据冗余和系统容量的情况下尽量重启最多的节点。而面试官希望从水平切分和垂直切分对比的角度考虑问题，没 Get 到点。（补课）
      • 上面的两个问题耗了好久，好像都没答到点上。

  • 好，我们来点简单的算法。

    • 给定 行 $M$ 列的二维数组表示 个节点，问能否从 到达 ， 每次只能往邻近节点走一格。其中数组中0表示该节点可通过，1表示不可通过。 比如：

      01000

      01010

      00010

      可到达。

      进阶：每次只能瞬移到相距为 格的位置，比如说当前位置是$(x, y)$，目标位置 必须满足

      • BFS。说了下思路，直接让写进阶的代码。啪啪啪啪啪…..

    • 一个奇数长度的数组。除了其中的一个数字只重复了一次，其他每个数字都重复两次，且重复的数字一定是成对的（也就是凑一块），找出不重复的那个数字。比如：1122334889900，不重复的是4。

      • 异或， 。面试官表示不够快，有没有更快的方法。
      • 想了两分钟（挖坑），找到了规律。从前向后查找下标为偶数的位置 ，满足条件： $a[i] \ne a[i + 1]$ ，满足条件的第一个 ， 就是不重复的那个数字，因对于 ，也一定满足 。所以可以做二分查找，复杂度变为 。

    • 最大子矩阵和：poj 1050 或 https://leetcode-cn.com/problems/max-submatrix-lcci/

      • 似乎是 ACM 的入门题。非ACMer，这题真没见过。直觉告诉我正解应该是 DP ，然而我不会（摔）….. （TwT）。
      • 提示：如果一时间想不出来，从暴力开始吧。说暴力的思路，口算复杂度。
      • 提示：求和能不能优化。
      • 好了，就这样吧，我们聊点别的（估计是时间不太够了）。

  • 简单问基础

    • 线程和进程的区别？
    • 多线程用过锁么？（怎么可能没用过？）
      • 没用 C++ 写过完整项目，没怎么直接调过 OS API 的锁。但 Go 写项目的时候用得挺多的，基本就是加锁加锁加锁加锁加锁加锁（面试官笑了）。。。。。。毕竟很多时候还是得 Share memory 嘛。
    • C++ 基础怎样？
      • 开始只会 C/C++，但后来写别的语言去了，这几年都没怎么用过，近来看 Ceph 源码接触得多。常见的特性应该还是记得的。
      • 了解 vector 么？没看过 STL 源码，但如果我去实现我会 blablabla（主要说了下扩容的情况还有要注意的问题）。糊弄过去了。
      • std::shared_ptr 的原理？利用 RAII ，没看过实现，如果我去实现会 blablabla，然后说了下要注意的线程安全问题。（感觉应该答得八九不离十，然后我想下一步是不是要问循环引用和 weak_ptr 了？）

  • 今天就这样吧。提问环节。

    • 对我的评价？
      • 技术面挺广，比如你能提到一致性 hash，能看出你的实践挺多的。但有时候太过拘泥于现有设计了，思路真的需要放开一些，毕竟我们大多情况下都是面对新的问题。
      • 算法 coding 慢了点。而且代码的风格 emmmm。然后跟我说了下那个题的代码应该怎样写会更清晰。（因为想尽快把题撸完，就在代码风格上有点掉以轻心了）
      • 反应挺快的。特别是数组那题。
      • 如果你在想过没过，那我肯定不能回答你的 (←_←)。
    • 能介绍一下组里做的事么 OwO ，我挺感兴趣的？
      • 我就不介绍了，如果有二面，让二面面试官来介绍吧，我怕我说得太 low 了，然后你就不想来了 (←_←)。

  • 刚好周五的晚上，互祝周末愉快。

面了两小时，面试体验很不错。之后下楼逛了一圈，觉得好些问题都没答到点。于是佛了：随缘吧。

后来发现其实匹配到了一位 ACM 退役的面试官。0_0

二面

二面面试官好像挺忙的，约了个早上的时间。然后我起了个大早。（啊，好困）

  • 你是在家考研？网易那边咋回事，怎么会想来腾讯？我：……

  • 自我介绍。（老问题了）….

  • 问 Linux 操作

    • 100 行 100 列 的文件，求50行30列内容。
    • 怎么看网络带宽。
    • 看磁盘的活动。

  • 听说过内存池吗，说说理解？

    • 听过这三个字，没深究过。听起来和分配器相关，就按分配器的思路答了（似乎是正确的）。主要解决的问题：避免频繁向操作系统要内存（大量系统调用，效率是很低的），高效分配小对象，防止内存碎片…….
    • 为了讲明白思路，在白板上简单瞎糊了个数据结构。

  • （既然你都开始了.jpg）手撕：实现一个4k小块的内存池，写一下伪代码就可以了。

    • Emmm…4k其实不算小…都…一个内存页了，Linux 都用 Buddy 系统去管了。
    • 说了最简单的实现思路：大块切小块，小块利用前 4 个 Bytes （64位系统上是 8 个 Bytes）作指针，串起来，形成一个 freelist。继续用伪代码来写，因为要直接操作内存指针，伪代码就越写越像 C，最后干脆用 C 来写了。大概写了 init、free、alloc、extend 四种操作。
    • 利用前 4 个 Bytes 作指针实现有点 Hacky，被说了这块很不清晰。当时觉得加点注释会好很多，后来想想用 union 更好。

  • 设计题：敏感词过滤。敏感词库有一千万个敏感词，每个敏感词不超过100个字符。系统要接收文章，检查文章里面是不是有敏感词。

    • 容量估计：先口算估计一下一个节点能不能装下这一千万个敏感词：能！。（要是不能，咱可以用一致性Hash，分个片🐶）
    • 匹配：方法不少，比较简单又实用的肯定就是 Trie 树了。
    • 写一个小服务，把这一千万个敏感词都加载到内存里面，构建这个敏感词库的 Trie 树。调用方把文章传过来，用这棵树对文章内容做匹配就好，效率应该不错。

  • 设计题：有十万个文件，文件名编号 1-100000。要求按照文件的前32个Bytes进行排序，然后将文件名重命名到其正确的顺序。设计一个尽可能快的方案。

  • 手撕代码：以粒度 反转链表。比如：

    要求原地算法，写完后必须编译通过。

    • 题不难。代码就不放了。

    • 编译通过后要求手写一些测试用例。注意覆盖边界条件。

  • 说一说你的优势和劣势。

  • 你是偏外向还是内向？

  • 问了问对我的评价。

二面并不难，面了一个小时左右。面试官希望我能早点入职，加了微信就匆匆结束了。
